## 0. Cadrage

### 0.1 Objectif

Audit produit, fonctionnel et production-ready de SmartVest, **page par page** + **transverse** (onboarding, brokers, banking, doc, sécurité). Livrer une checklist Go Live exploitable par une équipe design + dev + QA + ops.

### 0.2 Niveau cible

- **Bêta privée fermée** par défaut (`SAFE_PUBLIC_MODE=true`, simulation only, 5-50 users invités)
- **Ambition prod publique court terme** : audit conduit comme si on visait (c), mais distinction claire entre exigences bêta vs prod.

### 0.3 Périmètre

**INCLUS** : frontend (UX/parcours/états), backend (logique métier, garde-fous), DB (schéma, RLS), compliance (wording, MIFID, audit chain, RGPD).

**EXCLUS** : infra/devops, tests CI/CD (signalés si trous évidents).

### 0.4 Benchmarks

- **Trade Republic** — UX grand public, simplicité bancaire
- **Composer** — no-code/low-code quant, IA pilotée
- **TradingView** — cockpit analyse profond
- *Filigrane* : IBKR/Saxo pour exécution/connectivité

### 0.5 Méthodologie d'évaluation (grille 9 dimensions)

| Dim | Question |
|---|---|
| 1. Objectif métier | Pourquoi cet onglet existe ? |
| 2. Inputs | Quelles données entrent ? |
| 3. Calculs/sorties | Quels outputs affichés ? |
| 4. Actions user | Quels boutons/mutations ? |
| 5. États | Loading / vide / erreur / no-permission ? |
| 6. Dépendances | Brokers, EODHD, Claude, Supabase ? |
| 7. Sécurité/compliance | Wording, secrets, audit chain, MIFID ? |
| 8. Gaps prod-ready | Ce qui manque pour user externe |
| 9. Verdict | OK / Partiel / À concevoir / Critique |

## 1. P1 — Map 360°

*Voir le tableau complet en fin de section. Synthèse en haut.*

### 1.1 Synthèse

**38 pages** identifiées (pas 11 comme initialement listées). Architecture **bimodale** :

- **Core métier** (cash, funding, goals, imports, portfolio) : 🟢 solide, production-like
- **AI & analyse** (Lisa, backtest, optimizer, bot-lab, patterns) : 🟢 très avancé, bien câblé
- **Settings & config** (brokers, delegation, hyper-trading, sniper, strategy-mode) : 🟡 partiels, audit détaillé requis
- **Hubs d'agrégation** (history, performance, settings root) : 🟠 squelettes, par design ?

**Distribution finale** : 23 🟢 / 13 🟡 / 2 🟠 / 0 🔴

**Rouge majeur** : aucune page totalement cassée, MAIS de nombreux 🟢 cachent des gaps UX/onboarding (zero help tooltip, zero empty state guidé pour 1er user).

### 1.3 Priorisation P2 (deep dive critique)

Les **4 onglets prioritaires** pour P2 (parce qu'ils concentrent le risque produit/compliance) :

1. **`/lisa`** (1413 lignes) — cœur du produit, autopilot, decision engine
2. **`/settings/delegation`** (453 lignes) — mandats AUTONOMOUS_GUARDED, kill-switch
3. **`/settings/hyper-trading`** (421 lignes) — mode haute intensité, opt-in strict
4. **`/onboarding`** — premier contact user, MIFID-sensitive

Les **secondaires** pour P3 :

5-7. Cash/funding ecosystem (3 pages)

12-13. News + market-context
